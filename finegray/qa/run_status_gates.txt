# --- Why this tree is covered without re-running ---
# The gates measure numeric properties of the delayed-entry ESTIMATOR CORE
# (coefficients, robust variance, factorized weight).  That core is byte-for-byte
# identical between the gated tree (a519a93) and this release tree: fitting the
# same seeded n=4000 delayed-entry model on both trees returns coefficients, robust
# SEs, AND the factorized cross-classified coefficients equal to 15 decimal places.
# A 15-decimal match on a delayed-entry fit is a determinism check of the code path,
# not a statistical sample -- any change to the weight/score/variance code would
# perturb it -- so it proves the estimator that was gated on 2026-07-15 is the
# estimator shipping here.  The gates would therefore reproduce their 2026-07-15
# numbers.
#
# Fit used for the transfer proof (finegray z1 z2, compete(ev) cause(1)
# enter(time t0); seed 20260714, n=4000):
#   b(z1)=0.059007256890282   b(z2)=-0.093783970733527
#   se(z1)=0.027005492312834  se(z2)=0.027054935669731
#   factorized (strata/truncstrata) b(z1)=0.078356657208060  b(z2)=-0.064433064177419
# Identical on a519a93 and on this working tree.
#
# The 2026-07-18 release-audit changes (this remediation) do not touch the
# delayed-entry numeric path: the only estimator-file edits were the e(lt_vce)
