CRITICAL ISSUES (MUST FIX)
==========================

1. UNSAFE DATE CONVERSIONS (4 locations)
   Files: R/tvexpose.R (2), R/tvmerge.R (2)
   
   Problem: as.numeric() silently converts character dates to NA
   Example: "2020-01-01" → NA (with warning, but analysis continues!)
   
   Locations:
   • tvexpose.R:527-528   - Master entry/exit dates
   • tvexpose.R:558-559   - Exposure start/stop dates
   • tvmerge.R:618-619    - Dataset 1 dates
   • tvmerge.R:796-797    - Output date conversion
   
   Impact: Data integrity compromised, analysis results invalid
   Fix: Implement convert_to_numeric_date() helper + validation

2. EMPTY MASTER DATASET VALIDATION
   File: R/tvexpose.R (after line 420)
   Problem: Accepts 0-row master, produces 0-row output silently
   Impact: Undetected data loading errors
   Fix: Add nrow(master) > 0 check with clear error

3. DUPLICATE IDS IN MASTER DATASET
   File: R/tvexpose.R (after line 420)
   Problem: Same person can appear twice, violates design assumption
   Impact: Undefined behavior, double-counting of person-time
   Fix: Add anyDuplicated() check

4. ID TYPE MISMATCH DETECTION
   File: R/tvexpose.R (after line 420)
   Problem: numeric IDs vs character IDs cause silent join failure
   Impact: 0-row output with no error message
   Fix: Validate class(master_id) == class(exposure_id)

5. CARTESIAN PRODUCT MEMORY EXPLOSION
   File: R/tvmerge.R (lines 735-777)
   Problem: 50×50 periods/person = 2,500 rows; no warning before merge
   Impact: Out-of-memory crashes on large datasets
   Fix: Add estimate_cartesian_size() with warnings/errors

6. INFINITE DATE VALIDATION
   File: R/tvexpose.R (after line 420)
   Problem: Inf dates silently propagate through calculations
   Impact: All results become Inf/NaN
   Fix: Add is.infinite() check

7. ID TYPE MISMATCH IN tvmerge DATASETS
   File: R/tvmerge.R (after line 470)
   Problem: No common IDs across datasets → empty output
   Impact: Silent failure, user doesn't know why output is empty
   Fix: Add validate_overlapping_ids()

HIGH PRIORITY ISSUES (SHOULD FIX)
=================================

8. NA VALUES IN EXPOSURE COLUMN
   File: R/tvexpose.R (after line 420)
   Problem: NA exposure values create ambiguous categories
   Fix: Check and error on any(is.na(exposure_data[[exposure]]))

9. KEEPVARS VARIABLE VALIDATION
   File: R/tvexpose.R (after line 420)
   Problem: Typos in keepvars cause cryptic downstream errors
   Fix: Validate setdiff(keepvars, names(master)) is empty

10. CONFLICTING EXPOSURE TYPE PARAMETERS
    File: R/tvexpose.R (after line 420)
    Problem: evertreated=TRUE + currentformer=TRUE = undefined behavior
    Fix: Check only ONE exposure type parameter is set

    - Parameter conflict resolution undefined
    - Overlapping ID warning threshold undefined
    - Large dataset performance not documented
